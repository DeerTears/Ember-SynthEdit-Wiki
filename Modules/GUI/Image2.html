My mdzk

Image2 is an input-handling GUI element, that processes user input when clicking on the image.

Although it only appears to have the output of Frame Count, it serves as a callback to any PatchMemory node connected to its Animation Position value. This makes it really powerful at giving the user a control to click on while letting you personalize the way the user makes parameter changes to your synth.

It's unexpectedly strong for how unconventional it is.

Mouse Response

When the user clicks on this control, how should their input change the value of this sub-control?

off

Will not respond to the mouse. Used for background elements and decorations.

auto

Pulls from mouse_response property in the the .txt file associated with the image chosen. Same as off if not set.

vertical

Vertical mouse movement changes the value. Moving up increases the value, moving down decreases the value.

horizontal

Horizontal mouse movement changes the PatchMemory Value connected to this Image2 node. Moving right increases the value, moving left decreases the value.

clicks

Clicking and holding the mouse down on this button sets the value to Low. Releasing the mouse sets the Value to High for the associated PatchMemory node.

This is not the same as the Spring node. There is no reporting of inbetween values like you might see on a pitch bend wheel in a midi keyboard.

rotary

After clicking and holding the mouse, the mouse needs to be moved in a circular motion. Moving the mouse clockwise increases the value, moving it counter-clockwise decreases th value.
