My mdzk

A pre-made set of knobs, sliders, buttons, joysticks, and other input methods. Their is appearance determined by the current notes/Skins/index for the whole project.

These are the best starting point for a new user looking to give their VST users control over different parameters.

Image2 is an input-handling GUI element, that processes user input when clicking on the image.

Although it only appears to have the output of Frame Count, it serves as a callback to any PatchMemory node connected to its Animation Position value. This makes it really powerful at giving the user a control to click on while letting you personalize the way the user makes parameter changes to your synth.

It's unexpectedly strong for how unconventional it is.

Mouse Response

When the user clicks on this control, how should their input change the value of this sub-control?

off

Will not respond to the mouse. Used for background elements and decorations.

auto

Pulls from mouse_response property in the the .txt file associated with the image chosen. Same as off if not set.

vertical

Vertical mouse movement changes the value. Moving up increases the value, moving down decreases the value.

horizontal

Horizontal mouse movement changes the PatchMemory Value connected to this Image2 node. Moving right increases the value, moving left decreases the value.

clicks

Clicking and holding the mouse down on this button sets the value to Low. Releasing the mouse sets the Value to High for the associated PatchMemory node.

This is not the same as the Spring node. There is no reporting of inbetween values like you might see on a pitch bend wheel in a midi keyboard.

rotary

After clicking and holding the mouse, the mouse needs to be moved in a circular motion. Moving the mouse clockwise increases the value, moving it counter-clockwise decreases th value.

global.txt is a necessary file for every notes/Skins/index folder in SynthEdit. It describes the system fonts (or distributed fonts) to use when drawing text in the plugin.

Global Syntax

The syntax for global.txt is CSS1. It was recommended for use in 1996, revised in 2008, before being superseded on September 13th 2018 by CSS. However, global.txt still uses CSS1 rather than modern CSS. The original reference is here https://www.w3.org/TR/REC-CSS1/.

There are better learning resources for modern CSS these days than there ever were for CSS1. Here's the most important differences, everything else is basically fine if you know CSS.

CSS | CSS1 --- | -- ; | newline : | space curly brackets | put on the same line parenthesis | put on the same line #ffffff | #FFFFFF anything cool not mentioned here | don't even bother

There's no classes, but you can define unique FONT_CATEGORY selectors in parenthesis, and call upon it with any Style property on nodes found in SynthEdit, such as Text Entry4.

Everything else should be reletavely famliiar for a CSS programmer.

FONT_CATEGORY

Each font definition starts with FONT_CATEGORY followed by one of the following categories:

category | description --------------- | ---------- module_title | ?? panel_group | ?? control_label | ?? control_edit | ?? switch_label | LED Stack & Rotary Switch item labels selector_text | ?? tty | Container and VST plugin display text plug | Heading on properties screen

#todo See what each of these cateogires are in a more tangible way.

I recommend you copy the FONT_CATEGORY definitions from the global.txt of other notes/Skins/index folders, so you don't have to know for certain what each of these does, and keep using the same relative changes in font size between categories.

You can also define FONT_CATEGORY styles by writing a name in quotation marks. These styles can be used in the Text Entry4 node, under its Style property. Styles are not case-sensitive.

Examples styles are:

  • "properties_heading"
  • "Normal"
  • "Heading 1"
  • "Heading 2"
  • "Heading 3"

Each FONT_CATEGORY can have any number of the following properties:

font-family

Choose one or more font families to use by putting them in a comma separated list. These must be system fonts you can find on the user's computer, or otherwise distributed with the VST itself.

Example: Arial, Helvetica, "sans-Serif"

The comma separated list of families should end with the category it falls into as a fallback, in case none of the chosen families can be found.

Font FamilyCommon Replacements
"sans-serif"Arial, Helvetica
"serif"Times, Times New Roman
"cursive"Zapf-Chancery
"fantasy"Western
"monospace"Courier New

Each of the default fonts may be subject to limitations like only allowing certain font sizes, weights, and styles. You can prevent font fallbacks by (legally!!) providing the fonts yourself in the VST distribution and asking the user to install it, or by including the file in

SynthEdit's help page doesn't recommend distributing fonts with your VST for legal reasons, but you can download open source, free-to-distribute fonts on https://fonts.google.com/. Please do not re-distribute fonts in a way that breaks the terms of service from font creators and licensed distributors who provide these fonts.

font-size

How large in pts the font should be. Only takes integers.

Example: font-size 12

font-color

The color of the font in hexadecimal.

#todo Does this support web color names like White, Black, and Goldenrod?

Example: font-color #FFFFFF makes the text white. font-color #000000 for black, #00FF00 for green. Any hexadecimal combination is valid.

font-style

For fonts without many weights, you can occasionally choose a font style to make text look bold or italicized.

Examples

  • font-style normal
  • font-style italic

Styles not supported:

  • oblique

font-weight

An optional descriptor to select a specific font weight within a font family, if it's installed or provided. The most common choice is Bold. Other font weights distributed with a font family can be used if they're included, such as Medium, Light, Thin, Italic, Bold Italic, and Condensed. Some fonts come in numbered weights such as 100, 200, and 300. These are also valid inputs. Consult the font weight names of the font you're using and it should be usable here.

Examples:

  • font-weight bold
  • font-weight 400
  • font-weight light italic

Weights not supported:

  • bolder
  • lighter

vst-vertical-offset

Specify an offset in pixels when drawing text. This is ideally to compensate for changes in graphics API when exporting 64 bit plugins or VST3 versions of plugins.

Example: vst-vertical-offset 16

legacy-vertical-offset

Changes the height of the baseline on the text when used in a VST2 or 32bit versions of the plugin. This will override the value set in vst3-vertical-offset.

Example: legacy-vertical-offset 16

Skins are away to retexture the notes/Modules/Control/index nodes provided to you by SynthEdit.

Each skin needs the following:

  1. A folder in C:\Users\Public\Documents\SynthEdit Projects\skins
  2. A Global file describing which system font to use

I recommend copying the contents of an existing skin first into your new skin, to let you work on a project without making assets for all notes/Modules/Control/index nodes. Using a skin with missing Controls will create rendering issues.

To choose the skin for your project:

  1. Right click on the workspace.
  2. Click on Panel Edit...
  3. Right click on the Panel Edit workspace.
  4. Choose your skin from the Skin nested list.

If you do not see your skin listed, you still need to create a folder and ensure it has a global.txt with font definitions first. See Global for more information.

Choosing Skin -> View Skin Files will display the folder of your currently selected skin in your OS file browser.

Avoiding Cloned Modules

![[container clone.png]]

Adding a MIDI to CV makes a clone of the modules you're inputting data through.

This is why there can only be one MIDI to CV module per container. Using these increases your module count behind the scenes.

Modules outside of a container when exporting

Your VST export needs to be in a container, with no modules (except MIDI In, MIDI Out, Sound In, and Sound Out).

Voice Limits

Each container starts with 6 voices. You can increase it to 128 voices maximum using MIDI to CV.

Stands for Virtual Studio Technology.

VST is a trademark of Steinberg Media Technologies GmbH, registered in Europe and other countries.

Steinberg's Website

There are two standards of VSTs: VST2, and VST3.

VST2

Use the .dll suffix.

Steinberg stopped supporting the SDK for VST2 in 2013. VST2 plugins still work on many hosts, and are a safe bet for backwards compatibility.

VST3

Use the .vst3 suffix.

Can suspend itself when not in use, without requiring the plugin developer to implement this feature manually.

Organizes automation parameters into groups for easier browsing.

The GUI and window can be resized.

Uses unicode, making it easier to use text in languages with non-ASCII characters.

SynthEdit is a tool for creating VST plugins using a node-based visual interface. Created by Jeff McClintock in 1987, it's still in active development, and is responsible for several VSTs sold on the plugin market.

Some examples of plugins made with SynthEdit are Phasm by novaflash and KX-SYNTH-X16 by Claudia Kalensky.

As of 2022-01-17, the latest stable SynthEdit version is 1.4. See the 1.4 changelog to read more.
